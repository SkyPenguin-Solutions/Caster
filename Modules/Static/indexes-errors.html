Errors and General Guidence

    Error (You don't have permission to capture on that device (socket: Operation not permitted))
    • simply just run the program again as sudo
    Could not get a valid response from the server during GET request method, this may have been a connection issue, host issue etc
    • This means there was not a valid response and that the host you may have set could be wrong, ensure your host or target is set correctly
    Response code from URL was not 200, this may be due to connection issue, host issue, Timeout issues and more
    • The request to the given host and URL was not 200 which means enumeration or exploitation may have failed
    Could not create a new request for POST using http.NewRequest and with current client and set headers
    • This is a developer error, it means that when the program tried to create a new request for the URL with POST data either JSON or XML formats, the URL parser returned an error. This can be due to the use of wrongful symbols or data that is not a hostname or port entering that section
    Could not create and execute the current request for POST method using current data"
    • The request for the POST based method could not be executed, this can be due to host data, client data, client being down, server or host being down etc.
    It seems as if the program paniced, this may be due to an array issue. If you want tracebacks use --trace for this program. See error documentation for more information
    • This is a radical error and should never happen, if you have flags --errors and --trace this error and warning will pop up however if it does not its fine, this simply means that when the program was parsing hostnames, mac addresses etc during removing extra and duplicate values there was another address or MAC or even source that may have the same MAC address. For example if 10.0.0.1 and 10.0.0.5 have the same mac FF:FF:FF:Ff:FF:FF then this error will happen because to the program the MAC ias the same and is a duplicate value which can cause data manipulation and output errors. This is not really that bad or will crash the program but it can happen alot and output alot if 1 the flags are enabled or 2 the IP source or MAC source have even a single duplicate.
    Could not make a direct connection to the local server, the server must be down. If you want the server up ensure you set the flag --server=true before running the program
    • this error means that it could not make a connection to the web server, in other words when using `check server` the server responded with a code that was not 200, meaning the server is down or that you used the flag --server=false which means you disabled the server
